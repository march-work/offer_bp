# 城市居住成本换算器 — 算法完整拆解

&gt; 本文档提供不同城市间买房（新/二手）和租房（整租/合租）成本的标准化换算方法，供 Offer 决策时参考。

---

## 1. 设计动机

### 1.1 核心问题

不同城市间的居住成本差异巨大，仅凭房价或房租数字难以直观比较：
- 北京 500 万的房子 vs 成都 150 万的房子，哪个相对更"贵"？
- 上海整租 8000 元 vs 武汉合租 2000 元，如何换算成可比指标？
- 买新房 vs 买二手房 vs 租房，哪种方式在不同城市的相对压力更合理？

### 1.2 解法思路

**将绝对成本转化为相对指标，建立跨城市可比的标准化体系。**

核心思路：
1. 以"收入房价比"和"租金收入比"为核心相对指标
2. 使用各城市真实房价/收入数据（JSON）进行计算
3. 建立"居住压力指数"统一度量不同居住方式

---

## 3. 子公式详解 — 买房成本换算

### 3.1 基础参数定义

| 参数 | 说明 | 取值来源 |
|------|------|----------|
| `P_total` | 房屋总价 | 数据来源或用户输入 |
| `P_unit` | 单位面积房价（万元/㎡） | 数据来源 |
| `Area` | 房屋面积（㎡） | 用户输入（默认 90㎡） |
| `Income_year` | 年收入（万元） | 用户输入 |
| `Income_month` | 月收入（万元） | `Income_year / 12` |
| `Down_payment_ratio` | 首付比例 | 城市政策（默认 0.3） |
| `Loan_ratio` | 贷款比例 | `1 - Down_payment_ratio` |
| `Loan_years` | 贷款年限 | 用户选择（默认 30 年） |
| `Interest_rate` | 贷款利率 | 当前 LPR（默认 3.2%） |

### 3.2 核心指标计算

#### 3.2.1 总价收入比（核心指标）

```
总价收入比 = P_total / Income_year
```

**含义**：用多少年的总收入可以买这套房。

**评级标准**：
| 总价收入比 | 评级 | 说明 |
|-----------|------|------|
| &lt; 6 | 非常轻松 | 国际标准合理线 |
| 6 ~ 10 | 合理 | 国内普遍接受范围 |
| 10 ~ 15 | 偏高 | 有一定压力 |
| 15 ~ 20 | 沉重 | 压力很大 |
| &gt; 20 | 极重 | 难以承受 |

#### 3.2.2 首付压力

```
首付 = P_total × Down_payment_ratio
首付压力 = 首付 / Income_year
```

**含义**：需要攒多少年的收入才能付首付。

#### 3.2.3 月供计算（等额本息）

```
贷款总额 = P_total × Loan_ratio
月利率 = Interest_rate / 12
还款月数 = Loan_years × 12

月供 = 贷款总额 × [月利率 × (1+月利率)^还款月数] / [(1+月利率)^还款月数 - 1]
```

#### 3.2.4 月供收入比

```
月供收入比 = 月供 / Income_month
```

**含义**：每月收入中用于还房贷的比例。

**评级标准**：
| 月供收入比 | 评级 | 说明 |
|-----------|------|------|
| &lt; 0.3 | 轻松 | 银行理想线 |
| 0.3 ~ 0.5 | 合理 | 可接受范围 |
| 0.5 ~ 0.7 | 偏高 | 生活质量受影响 |
| &gt; 0.7 | 沉重 | 影响基本生活 |

### 3.3 新房 vs 二手比较

```
新房溢价率 = (P_unit_new - P_unit_secondhand) / P_unit_secondhand
```

**考虑因素**：
- 新房：装修成本、期房等待时间、税费优惠
- 二手：即买即住、装修省时间、学区确定性

## 4. 子公式详解 — 租房成本换算

### 4.1 基础参数定义

| 参数 | 说明 | 取值来源 |
|------|------|----------|
| `Rent_month` | 月租金（元） | 数据来源或用户输入 |
| `Rent_unit` | 单位面积月租金（元/月/㎡） | 数据来源 |
| `Area_rent` | 租房面积（㎡） | 用户输入（默认 60㎡整租 / 20㎡合租） |
| `N_people` | 居住人数 | 用户输入（整租默认 2 人 / 合租默认 1 人） |

### 4.2 核心指标计算

#### 4.2.1 租金收入比（核心指标）

```
租金收入比 = Rent_month / Income_month
```

**含义**：每月收入中用于房租的比例。

**评级标准**：
| 租金收入比 | 评级 | 说明 |
|-----------|------|------|
| &lt; 0.25 | 轻松 | 理想状态 |
| 0.25 ~ 0.35 | 合理 | 可接受范围 |
| 0.35 ~ 0.5 | 偏高 | 压力较大 |
| &gt; 0.5 | 沉重 | 影响生活质量 |

#### 4.2.2 单位面积租金

```
单位面积租金 = Rent_month / Area_rent
```

**用途**：跨城市比较每平米居住成本。

#### 4.2.3 人均租金

```
人均租金 = Rent_month / N_people
```

**用途**：合租场景下的个人成本衡量。

### 4.3 整租 vs 合租比较

```
合租节省比例 = (整租人均 - 合租人均) / 整租人均
```

**考虑因素**：
- 整租：隐私、空间、自由
- 合租：省钱、社交、有人分担

## 5. 子公式详解 — 跨城市标准化换算

### 5.1 数据来源

城市居住成本数据直接从 JSON 文件加载，不再使用硬编码城市分级系数。每个城市包含：

- 各区县新房/二手房均价（万元/㎡）
- 各区县整租/合租单价（元/月/㎡）
- 城市人均可支配收入和消费支出（元/年）
- 分行业平均薪资

数据文件位于 `public/cities/{城市名}/` 下，通过 `city-data.ts` 加载。

### 5.2 标准化压力指数

为了统一度量不同城市、不同居住方式的压力，设计"居住压力指数"：

```
居住压力指数 = (总价收入比 / 10) × 0.5 
             + (租金收入比 / 0.3) × 0.3
             + 城市生活成本系数 × 0.2
```

**说明**：
- 总价收入比除以 10：将 10 年作为基准（压力指数=0.5）
- 租金收入比除以 0.3：将 30% 作为基准（压力指数=0.3）
- 权重：买房 0.5，租房 0.3，城市生活 0.2

**压力指数评级**：
| 压力指数 | 评级 | 颜色 |
|---------|------|------|
| &lt; 0.5 | 轻松 | 🟢 绿色 |
| 0.5 ~ 0.8 | 合理 | 🟡 黄色 |
| 0.8 ~ 1.2 | 偏高 | 🟠 橙色 |
| &gt; 1.2 | 沉重 | 🔴 红色 |

### 5.3 城市换算示例

**场景**：北京 vs 成都，年收入 30 万

**北京**：
```
二手房总价：500 万（90㎡ × 5.56 万/㎡）
总价收入比 = 500 / 30 = 16.7
租金（整租）：8000 元/月
租金收入比 = 8000 / 25000 = 0.32
```

**成都**：
```
二手房总价：150 万（90㎡ × 1.67 万/㎡）
总价收入比 = 150 / 30 = 5.0
租金（整租）：3000 元/月
租金收入比 = 3000 / 25000 = 0.12
```

**换算结论**：同样年收入 30 万，成都的居住压力指数 0.53，北京 1.275，成都是北京的 41.6%。

## 6. 买房 vs 租房 决策换算

### 6.1 年成本比较法

将买房和租房都换算成"年居住成本"进行比较：

**买房年成本**：
```
买房年成本 = 首付机会成本 + 月供 - 本金偿还 + 物业费 + 折旧
```

其中：
- 首付机会成本 = 首付 × 理财收益率（默认 3%）
- 本金偿还 = 月供 - 贷款总额 × 月利率（首月近似）
- 物业费 = 3 元/月/㎡ × Area × 12
- 折旧 = P_total × 0.02（每年折旧 2%）

**租房年成本**：
```
租房年成本 = Rent_month × 12 × (1 + 租金涨幅)^年数
```

租金涨幅默认 3%/年。

### 6.2 回本时间计算

```
累计成本平衡点 = （买房累计成本 - 租房累计成本）= 0 的年份
```

### 6.3 决策矩阵

| 场景 | 建议 |
|------|------|
| 总价收入比 &lt; 8，计划居住 &gt; 5 年 | 优先买房 |
| 总价收入比 8~15，计划居住 3~5 年 | 综合考虑 |
| 总价收入比 &gt; 15，或计划居住 &lt; 3 年 | 优先租房 |
| 月供收入比 &gt; 0.5 | 建议租房或降低标准 |

## 7. 完整计算示例（逐步演示）

### 场景：杭州互联网人 vs 成都互联网人

#### 输入参数

**共同参数**：
```
年收入：36 万（月收入 3 万）
目标面积：90㎡（买房）/ 60㎡（整租）
贷款：首付 30%，30 年，利率 3.2%
```

**杭州**：
```
二手房价：4.5 万/㎡ → 总价 405 万
整租租金：6000 元/月
```

**成都**：
```
二手房价：1.8 万/㎡ → 总价 162 万
整租租金：2800 元/月
```

---

#### 第 ① 步：杭州买房成本

```
① 总价收入比 = 405 / 36 = 11.25 → 评级「偏高」🟠

② 首付 = 405 × 0.3 = 121.5 万
   首付压力 = 121.5 / 36 = 3.38 年

③ 贷款总额 = 405 × 0.7 = 283.5 万
   月利率 = 3.2% / 12 = 0.267%
   还款月数 = 360
   月供 = 283.5万 × [0.00267×(1.00267)^360] / [(1.0035)^360 - 1]
        ≈ 12,300 元/月

④ 月供收入比 = 12,300 / 30,000 = 0.41 → 评级「合理」🟡
```

---

#### 第 ② 步：杭州租房成本

```
① 租金收入比 = 6,000 / 30,000 = 0.20 → 评级「轻松」🟢

② 单位面积租金 = 6,000 / 60 = 100 元/月/㎡
```

---

#### 第 ③ 步：成都买房成本

```
① 总价收入比 = 162 / 36 = 4.5 → 评级「非常轻松」🟢

② 首付 = 162 × 0.3 = 48.6 万
   首付压力 = 48.6 / 36 = 1.35 年

③ 贷款总额 = 162 × 0.7 = 113.4 万
   月供 ≈ 5,600 元/月

④ 月供收入比 = 5,600 / 30,000 = 0.187 → 评级「轻松」🟢
```

---

#### 第 ④ 步：成都租房成本

```
① 租金收入比 = 2,800 / 30,000 = 0.093 → 评级「轻松」🟢

② 单位面积租金 = 2,800 / 60 = 46.7 元/月/㎡
```

---

#### 第 ⑤ 步：居住压力指数对比

**杭州**：
```
Pressure_HZ = (11.25/10)×0.5 + (0.20/0.3)×0.3
            = 0.5625 + 0.20
            = 0.7625 → 评级「合理」🟡
```

**成都**：
```
Pressure_CD = (4.5/10)×0.5 + (0.093/0.3)×0.3
            = 0.225 + 0.093
            = 0.318 → 评级「轻松」🟢
```

**换算比例**：成都压力是杭州的 0.318 / 0.7625 ≈ 42%

## 11. 边界情况与注意事项

### 11.1 边界情况

| 场景 | 处理方式 |
|------|----------|
| 房价数据缺失 | 使用全市均价作为 fallback |
| 收入为 0 | 提示输入有效收入 |
| 比较城市相同 | 只显示绝对指标，不显示换算 |
| 贷款年限过短导致月供极高 | 高亮提示月供压力 |

### 11.2 数据时效性

- 房价数据每月更新
- 建议显示"数据更新时间"
- 系数表每年 review 一次

### 11.3 与 Offer 评测器的集成

本换算器可以作为 Offer 评测器的**补充模块**：
- 应届生评测器：在"Offer城市"旁增加"居住成本预览"
- 跳槽评测器：在"当前城市 vs 新城市"中增加居住成本对比

| 压力指数 | 评级 | 颜色 |

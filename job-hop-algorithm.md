# 跳槽决策评测器 — 算法完整拆解

> 本文档是 `development-report.md` 第四部分的详细展开，独立成册，供开发者实现时参照。

---

## 1. 设计动机

### 1.1 核心问题

跳槽者面对新 Offer 时，数字上的差距（比如年薪多了 5 万）往往掩盖了隐性变化：公积金少了、通勤远了、试用期有风险、环境变差了——到底划不划算？

### 1.2 解法思路

**对两个"工作状态"分别打分，做差得到净增量，再扣除转换成本。**

```
跳槽决策分 = Score_new - Score_current - 转换成本惩罚
```

- `Score_new` 和 `Score_current` 使用统一的单岗评分公式计算
- 转换成本是本评测器的**核心创新**——把模糊的"跳槽代价"量化为具体数字

---

## 2. 最终确定公式

```
跳槽决策分 = Score_new - Score_current - 转换成本惩罚
```

### 2.1 公式结构拆解

```
┌──────────── 单岗评分（两份工作各自算）────────────┐
│                                                  │
│  Score(job) = (dailySalary × envFactor)          │
│              / (35 × effectiveHours              │
│                × education × experience)         │
│                                                  │
│  统一单岗评分公式                                │
└──────────────────────────────────────────────────┘
                         +
┌──────────── 转换成本惩罚 ────────────┐
│                                     │
│  = 财务成本惩罚 + 风险惩罚           │
│  财务成本惩罚 ∈ [0, ~2.0]           │
│  风险惩罚 ∈ [0, 0.5]               │
└─────────────────────────────────────┘
                         ↓
┌──────────── 最终决策 ──────────────────┐
│                                      │
│  Δ = Score_new - Score_current       │
│  Δ_net = Δ - 转换成本惩罚            │
│                                      │
│  Δ_net > 0.5  → 赶紧跳              │
│  Δ_net > 0.2  → 值得跳              │
│  Δ_net > 0    → 可以跳              │
│  Δ_net > -0.2 → 差不多              │
│  Δ_net > -0.5 → 没必要              │
│  Δ_net ≤ -0.5 → 别跳                │
└──────────────────────────────────────┘
```

## 3. 子公式详解 — 单岗评分

### 3.1 单岗评分公式

对当前工作和新 Offer，各自独立计算：

```
Score(job) = (dailySalary × envFactor) / (35 × effectiveHours × education × experience)
```

所有子公式在两份工作上的计算逻辑完全一致，包括 PPP 标准化、年工作日、环境系数、教育系数、经验倍数。本项目仅面向中国大陆。

### 3.2 子公式快速回顾

#### dailySalary — 标准化日薪

```
dailySalary = (年薪 × PPP_factor) / 实际年工作日数
```

#### envFactor — 环境系数

```
envFactor = 工作环境 × 领导关系 × 同事关系 × 城市生活系数 × 食堂系数
```

5 个子因子取值表见应届生算法文档。

#### effectiveHours — 有效工时

```
effectiveHours = workHours + commute × officeRatio × shuttleFactor - 0.5 × restTime
```

其中：
- `officeRatio = (每周工作天数 - WFH天数) / 每周工作天数`
- `shuttleFactor`：无班车=1.0，有班车=0.3

#### education — 教育系数

| 学历 | 系数 |
|------|------|
| 专科及以下 | 0.8 |
| 本科 | 1.0 |
| 硕士 | 1.2 |
| 博士 | 1.4 |

> 与应届生评测器不同，跳槽评测器**保留**教育系数，因为单岗评分公式中它在分母，用于调节评分基准。

#### experience — 经验倍数

```
experience = 1 + (工作年限中位数 - 1) × 工作类型系数
```

| 工作年限 | 中位数 |
|----------|--------|
| < 1 年 | 0.5 |
| 1-3 年 | 2.0 |
| 3-5 年 | 2.7（文档中使用的值，可能对应中位数 3.7） |
| 5-10 年 | 7.5 |
| 10+ 年 | 15.0 |

| 工作类型 | 系数 |
|----------|------|
| 体制内（公务员/事业编） | 0.6 |
| 国企 | 0.8 |
| 外企 | 1.0 |
| 私企 | 1.2 |
| 创业公司 | 1.5 |

> **实现提示**：经验倍数的具体数值和计算方式以 `development-report` 中的示例数值为准。

#### 常数 35

"理想时薪基准"。含义：35 是一个标准化常数，代表"理想的工作性价比基准"。

## 4. 子公式详解 — 转换成本惩罚（核心创新）

### 4.1 总体结构

```
转换成本惩罚 = 财务成本惩罚 + 风险惩罚
```

| 成分 | 取值范围 | 说明 |
|------|----------|------|
| 财务成本惩罚 | [0, ~2.0] | 跳槽造成的经济损失标准化 |
| 风险惩罚 | [0, 0.5] | 试用期不通过的概率惩罚 |
| **总转换成本** | **[0, ~2.5]** | 两者相加 |

### 4.2 财务成本惩罚

```
财务成本惩罚 = (总财务成本 / 当前年薪) × 2
```

**设计逻辑**：
1. 计算所有跳槽造成的财务损失总额
2. 除以当前年薪得到"损失占年薪的比例"（0~1+）
3. 乘以 2 放大影响，使损失在决策分中权重合理

```
总财务成本 = 未兑现奖金 + 未归属股票 + 空窗期损失 + 福利年差额
```

#### 4.2.1 各项详细计算

| 成本项 | 计算方式 | 示例 | 说明 |
|--------|----------|------|------|
| **未兑现奖金** | 未发放的年终奖/绩效奖金金额 | ¥30,000 | 直接输入金额 |
| **未归属股票** | 未 vest 的 RSU/期权估值 | ¥37,500 | 用户自估值，0=无 |
| **空窗期损失** | (空窗天数 / 365) × 当前年薪 | ¥11,507 | 按日折算 |
| **福利年差额** | (公积金差 + 补贴差) × 12 | ¥24,000 | 年度总差额 |

#### 4.2.2 空窗期损失计算

```
空窗期损失 = (空窗天数 / 365) × 当前年薪
```

| 空窗期选项 | 天数 | 示例（当前年薪 ¥30万） |
|-----------|------|------------------------|
| 0 天 | 0 | ¥0 |
| 1 周 | 7 | ¥5,753 |
| 2 周 | 14 | ¥11,507 |
| 1 个月 | 30 | ¥24,658 |
| 2 个月 | 60 | ¥49,315 |
| 3 个月+ | 90+ | ¥73,973+ |

#### 4.2.3 福利年差额计算

```
福利年差额 = 公积金年差额 + 补贴差额

公积金年差额 = |旧公积金比例 - 新公积金比例| × 缴存基数 × 2 × 12
  // × 2 因为公积金 = 个人 + 公司各缴一份
  // × 12 因为按年计算

补贴差额 = 年度补贴总额差（用户直接输入，正数=新公司多，负数=新公司少）
```

**公积金差额示例**：

```
旧公司：公积金比例 12%，缴存基数 ¥25,000
新公司：公积金比例 8%，缴存基数 ¥25,000

公积金年差额 = |0.12 - 0.08| × 25,000 × 2 × 12
             = 0.04 × 25,000 × 2 × 12
             = ¥24,000/年
```

> **注意**：此为年差额，跳槽前只损失已工作月份对应的差额，此处简化为全额计入。即按年度总差额计算，不按月折算。

#### 4.2.4 财务成本惩罚完整示例

```
未兑现奖金 = ¥25,000
未归属股票 = ¥0
空窗期损失 = 14/365 × ¥300,000 = ¥11,507
福利年差额 = ¥24,000（公积金 12%→8%）

总财务成本 = 25,000 + 0 + 11,507 + 24,000 = ¥60,507
当前年薪 = ¥300,000

财务成本惩罚 = 60,507 / 300,000 × 2 = 0.403
```

### 4.3 风险惩罚

```
风险惩罚 = P(试用期不通过) × 0.5
```

| 试用期风险等级 | 概率 P | 风险惩罚 | 说明 |
|---------------|--------|----------|------|
| 低 | 0.05 | 0.025 | 大厂/稳定行业 |
| 中 | 0.15 | 0.075 | 一般公司 |
| 高 | 0.30 | 0.150 | 小公司/创业公司/试用期长 |

**设计逻辑**：
- 最高 0.5 分的惩罚（P=1.0 时）
- 实际上 P=0.30（高风险）已给出 0.15 的显著惩罚
- 对于 P=0.05（大厂），惩罚仅 0.025，影响很小

### 4.4 转换成本惩罚汇总

```
转换成本惩罚 = 财务成本惩罚 + 风险惩罚
             = 0.403 + 0.025
             = 0.428
```

## 5. 评级体系

| 跳槽决策分 | 评级 | 颜色（Tailwind） | 建议 |
|-----------|------|------------------|------|
| > 0.5 | 赶紧跳 | `text-green-700` | 新 offer 远优于当前，果断接受 |
| 0.2 ~ 0.5 | 值得跳 | `text-green-500` | 新 offer 明显更好，建议接受 |
| 0 ~ 0.2 | 可以跳 | `text-blue-500` | 新 offer 略好，可以考虑 |
| -0.2 ~ 0 | 差不多 | `text-yellow-500` | 差距不大，看个人偏好 |
| -0.5 ~ -0.2 | 没必要 | `text-orange-500` | 当前工作更好，建议留下 |
| ≤ -0.5 | 别跳 | `text-red-500` | 新 offer 明显更差，强烈建议留下 |

### 5.1 与应届生评测器评级的关键差异

| 维度 | 应届生评测器 | 跳槽评测器 |
|------|-------------|-----------|
| Score 含义 | 绝对倍数（你拿到 / 你应拿） | 增量值（新 - 旧 - 成本） |
| Score=0 | 低于市场 100% | 新旧完全一样（扣除成本后） |
| Score=1.0 | 等于市场期望 | 新的比旧的高 1 分（非常显著） |
| 评级方向 | 向上为好（>1 好） | 向上为好（>0 就该跳） |
| 负数 | 不可能出现 | 正常出现（不应该跳） |


## 7. 完整计算示例（逐步演示）

### 场景：北京互联网人跳槽到杭州

#### 输入参数

**个人信息**：
```
学历: 本科双非 (education = 1.0)
工作年限: 3-5年 (experience 中位数 ≈ 2.7)
工作类型: 私企 (工作类型系数 = 1.0)
```

**当前工作**：
```
年薪: ¥300,000 | 城市: 北京（一线）
每周: 5天, WFH: 2天
年假: 10天, 法定假日: 13天, 带薪病假: 5天
日均工时: 10h, 通勤: 1.5h, 休息: 1.5h
工作环境: CBD (1.1), 城市生活系数: 一线 (0.70)
领导: 中规中矩 (1.0), 同事: 萍水相逢 (1.0)
无班车、无食堂
```

**新 Offer**：
```
年薪: ¥380,000 | 城市: 杭州（一线）
每周: 5天, WFH: 2天
年假: 10天, 法定假日: 13天, 带薪病假: 5天
日均工时: 9h, 通勤: 1h, 休息: 1h
工作环境: CBD (1.1), 城市生活系数: 一线 (0.70)
领导: 善解人意 (1.1), 同事: 和睦 (1.1)
有食堂不错 (1.1), 无班车
```

**转换成本**：
```
未发年终奖: ¥25,000
未归属股票: ¥0
空窗期: 2周
试用期风险: 低 (P=0.05)
公积金: 12%→8%, 基数 ¥25,000
补贴差异: ¥0
```

---

#### 第 ① 步：计算当前工作 Score

```
① 年工作日 = 52 × 5 - (10 + 13 + 5 × 0.6)
           = 260 - 26
           = 234 天

② 日薪 = 300,000 / 234 = ¥1,282.05

③ officeRatio = (5 - 2) / 5 = 0.6
   有效通勤 = 1.5 × 0.6 × 1.0 = 0.9h
   effectiveHours = 10 + 0.9 - 0.5 × 1.5 = 10.15h

④ 环境系数 = 1.1 × 1.0 × 1.0 × 0.70 × 1.0 = 0.77

⑤ education = 1.0 (本科双非)
   experience = 1 + (2.7 - 1) × 1.0 = 2.7

⑥ Score_current = (1282.05 × 0.77) / (35 × 10.15 × 1.0 × 2.7)
                = 987.18 / 959.18
                ≈ 1.029
```

---

#### 第 ② 步：计算新 Offer Score

```
① 年工作日 = 52 × 5 - (10 + 13 + 5 × 0.6)
           = 260 - 26
           = 234 天
           （年假、WFH 等与当前工作一致）

② 日薪 = 380,000 / 234 = ¥1,623.93

③ officeRatio = (5 - 2) / 5 = 0.6
   有效通勤 = 1.0 × 0.6 × 1.0 = 0.6h
   effectiveHours = 9 + 0.6 - 0.5 × 1 = 9.1h

④ 环境系数 = 1.1 × 1.1 × 1.1 × 0.70 × 1.1 = 1.0287

⑤ education = 1.0 (同一个人)
   experience = 2.7 (同一个人)

⑥ Score_new = (1623.93 × 1.0287) / (35 × 9.1 × 1.0 × 2.7)
            = 1670.40 / 860.85
            ≈ 1.941
```

---

#### 第 ③ 步：计算转换成本

```
① 财务成本:
   未兑现奖金 = ¥25,000
   未归属股票 = ¥0
   空窗期损失 = (14/365) × 300,000 = ¥11,507
   公积金差额 = |0.12 - 0.08| × 25,000 × 2 × 12 = ¥24,000
   总财务成本 = 25,000 + 0 + 11,507 + 24,000 = ¥60,507

② 财务成本惩罚 = 60,507 / 300,000 × 2 = 0.403

③ 风险惩罚 = 0.05 × 0.5 = 0.025

④ 转换成本惩罚 = 0.403 + 0.025 = 0.428
```

---

#### 第 ④ 步：计算跳槽决策分

```
Δ = Score_new - Score_current
  = 1.941 - 1.029
  = 0.912

Δ_net = Δ - 转换成本惩罚
      = 0.912 - 0.428
      = 0.484

→ 评级「值得跳」🟢（0.2 < 0.484 ≤ 0.5）
```

---

#### 第 ⑤ 步：如果转换成本更高？

假设未归属股票为 ¥50,000，公积金从 12% 降到 5%：

```
公积金差额 = |0.12 - 0.05| × 25,000 × 2 × 12 = ¥42,000
总财务成本 = 25,000 + 50,000 + 11,507 + 42,000 = ¥128,507
财务成本惩罚 = 128,507 / 300,000 × 2 = 0.857

转换成本惩罚 = 0.857 + 0.025 = 0.882

Δ_net = 0.912 - 0.882 = 0.030
→ 评级「可以跳」🔵（0 < 0.030 ≤ 0.2）
```

**结论**：虽然新 Offer 的单岗评分高了将近一倍，但高额的转换成本（股票 + 公积金差额）几乎吃掉了全部增量。建议用户慎重考虑。

## 8. 单岗评分公式 vs 跳槽决策公式

| 维度 | 单岗评分 | 跳槽决策 |
|------|---------|---------|
| 公式 | 单一 Score | Score_new - Score_old - 成本 |
| 输入 | 一份工作信息 | 两份工作信息 + 转换成本 |
| 输出 | 绝对分值（0~5+） | 增量值（-2.5~5+） |
| 参考基准 | 常数 35 | 当前工作的 Score |
| 评级含义 | 工作性价比如何 | 这槽该不该跳 |
| 新增模块 | — | 转换成本惩罚 |

### 8.1 跳槽评测器复用的子公式

| 模块 | 说明 |
|------|------|
| `Score(job)` 公式 | 两份工作各自使用相同公式 |
| PPP 标准化 | 相同逻辑 |
| 年工作日计算 | 相同逻辑 |
| 环境系数 | 相同逻辑（5 个子因子） |
| 有效工时 | 相同逻辑 |
| 教育系数 | 相同逻辑 |
| 经验倍数 | 相同逻辑 |
| 常数 35 | 相同逻辑 |

## 10. 边界情况与注意事项

### 10.1 边界情况

| 场景 | 处理方式 |
|------|----------|
| 两份工作完全一样 | Score_new = Score_current → Δ=0，只看转换成本 |
| 年薪差很大但工时也差很大 | Score 差可能很小，因为单岗评分已经包含工时惩罚 |
| 转换成本 > 分数增量 | Δ_net < 0 → 建议"别跳"或"没必要" |
| 新公司公积金更高 | 福利差额为 0（只计损失，不计收益） |
| 没有当前工作（裸辞） | 需要特殊处理：Score_current = 0 或让用户输入"预估" |

### 10.2 转换成本的方向性

转换成本**只计损失，不计收益**。即：
- 公积金从 12%→8% = 损失 ✓ 计入
- 公积金从 8%→12% = 收益 → **不计入**（已经在 Score_new 的薪资中体现）
- 年终奖没发 = 损失 ✓ 计入
- 新公司年终奖更多 = 收益 → **不计入**（已经在 Score_new 中体现）

### 10.3 Score 与应届生评测器 Score 的可比较性

跳槽评测器的 Score 是单岗绝对分值，应届生评测器的 Score 是相对倍数。**两者不可直接比较。**

但跳槽评测器内部的 Score_new 和 Score_current 是**可比的**（同一个公式，不同输入）。

### 10.4 新 Offer Score 比旧工作高很多但决策分很低

这是正常且合理的情况。例如：
- 新 Offer Score = 3.0（很好）
- 当前 Score = 1.0（一般）
- 但转换成本 = 2.5（股票 + 公积金 + 空窗期 = ¥15万损失）
- 决策分 = 3.0 - 1.0 - 2.5 = -0.5 → "没必要跳"

这个结果在财务上是合理的：虽然新工作性价比更高，但你损失了太多既有利益，需要很长时间才能"回本"。

### 10.5 回本周期计算（可选功能）

报告中可以计算"回本周期"：

```
回本月数 = 总财务成本 / 月薪资差额
月薪资差额 = (新年薪 - 旧年薪) / 12

示例：
总财务成本 = ¥60,507
月薪资差额 = (380,000 - 300,000) / 12 = ¥6,667
回本月数 = 60,507 / 6,667 ≈ 9.1 个月
```

即跳槽后约 9 个月可以"回本"转换成本。

## 11. 与应届生评测器的算法对比

| 维度 | 应届生评测器 | 跳槽评测器 |
|------|-------------|-----------|
| **核心公式** | `日薪×环境×8 / (期望日薪×工时)` | `Score_new - Score_old - 成本` |
| **基准** | 动态期望日薪（学历×学校×城市×行业） | 常数 35 |
| **教育系数** | 融入期望薪资 | 分母独立计算 |
| **经验倍数** | 不需要（应届生=0） | 分母独立计算 |
| **工时惩罚** | 通过 `effectiveHours` 显式体现 | 通过 `effectiveHours` 显式体现 |
| **PPP** | 用于日薪标准化（仅中国大陆） | 用于两份工作的日薪标准化（仅中国大陆） |
| **输入复杂度** | 单表单 | 双表单 + 转换成本表单 |
| **Score 含义** | 你拿到 / 你应拿 | 新的 - 旧的 - 成本 |
| **Score=1** | 等于市场期望 | 新比旧高 1 分 |
| **评级数量** | 7 级 | 6 级 |
| **创新点** | 动态期望薪资模型 | 转换成本量化 |

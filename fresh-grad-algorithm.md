# 应届生 Offer 评测器 — 算法完整拆解

> 本文档是 `development-report.md` 第三部分的详细展开，独立成册，供开发者实现时参照。

---

## 1. 设计动机

### 1.1 核心问题

应届生没有工作经验，面对一个 Offer 时缺乏参照系——不知道这个数字在市场上处于什么水平。

传统的工作价值评估公式 `Score = 收益 / 付出` 中，分母包含教育系数和经验倍数。但应届生经验=0 时分母偏小，导致评分虚高，不适合直接使用。

### 1.2 解法思路

**把"参照系"从常数变成动态计算值。**

- 传统公式：分子是你的收益，分母是常数 35 × 工时 × 教育 × 经验 → 绝对分值
- 应届生评测器：分子是你的收益，分母是**你应该拿到的** → 相对倍数

Score = 1.0 意味着你刚好拿到"你的背景应该拿到的水平"。

---

## 2. 最终确定公式

```
FreshGrad Score = (dailySalary × envFactor × 8) / (expectedDailySalary × effectiveHours)
```

### 2.1 公式结构拆解

```
┌─────────────────────────── 分子 ───────────────────────────┐
│  dailySalary × envFactor × 8                              │
│  你实际获得的（日薪 × 环境加分 × 标准工时 8h）               │
└────────────────────────────────────────────────────────────┘
                            ÷
┌─────────────────────────── 分母 ───────────────────────────┐
│  expectedDailySalary × effectiveHours                      │
│  你应该获得的（期望日薪 × 实际付出的工时）                     │
└────────────────────────────────────────────────────────────┘
```

### 2.2 各项含义

| 变量 | 含义 | 计算来源 |
|------|------|----------|
| `dailySalary` | 实际标准化日薪 | `(年薪 × PPP标准化) / 实际年工作日数` |
| `envFactor` | 工作环境综合系数 | 5 个子因子相乘 |
| `8` | 标准工时常数 | 归一化：将日薪视角统一到 8h 标准工时 |
| `expectedDailySalary` | 市场期望日薪 | `期望年薪 / 260`（260 = 52周×5天） |
| `effectiveHours` | 每日有效工时 | `workHours + 通勤 - 休息` |

### 2.3 为什么乘以 8？

传统公式的分子是 `dailySalary`，分母中有 `effectiveHours`。但应届生的分母是 `expectedDailySalary`（也是日薪级别），如果分子分母都是日薪，工时因素就消失了。

乘以 8 的作用：**把日薪视角统一到"标准 8h 工时"，让分子分母的工时维度对齐。**

直观理解：
- 分子 = `日薪 × 8` = 你的 8h 标准工时对应的报酬
- 分母 = `期望日薪 × 有效工时` = 你的背景该拿的时薪 × 你实际付出的工时
- Score > 1 = 你每付出一个有效工时，获得的报酬超过了你的背景应该拿的水平

## 3. 子公式详解

### 3.1 dailySalary — 实际标准化日薪

```
dailySalary = (年薪 × PPP_factor) / 实际年工作日数
```

#### 3.1.1 PPP 标准化

PPP（Purchasing Power Parity）用于跨国比较。将不同国家的薪资转换为购买力等价金额。

PPP 因子查询表（数据来源见 `ppp-factors.ts`）：

| 国家/地区 | PPP 因子 | 说明 |
|-----------|----------|------|
| 中国 | 4.19 | 1 元人民币 ≈ 4.19 分（购买力折算） |
| 美国 | 1.0 | 基准 |
| 日本 | 1.2 | |
| ... | ... | 完整表见 `ppp-factors.ts` |

**注意**：对于中国市场内的比较，PPP 因子主要影响跨国用户的对比结果。同一国家内不同城市间的差异由"城市系数"处理。

#### 3.1.2 实际年工作日数

```
实际年工作日 = 52 × 每周工作天数 - (年假 + 法定假日 + 带薪病假 × 0.6)
```

**参数说明**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `每周工作天数` | 选择 | 5 | 4/5/6/7 |
| `年假` | 数字 | 5 | 天 |
| `法定假日` | 数字 | 13 | 天（中国标准） |
| `带薪病假` | 数字 | 3 | 乘以 0.6 因为并非 100% 利用 |

**计算示例**：
```
52 × 5 - (5 + 13 + 3 × 0.6) = 260 - 19.8 = 240.2 天
```

#### 3.1.3 完整示例

```
年薪 = ¥220,000, PPP = 4.19, 年工作日 = 240.2
dailySalary = (220,000 × 4.19) / 240.2 = ¥3,836.39
```

> **实现提示**：PPP 标准化可能采用简化处理（直接用 PPP 做除数或其他方式）。实现时需确认项目统一的 PPP 处理方式并保持一致。

```
envFactor = 工作环境 × 领导关系 × 同事关系 × 城市生活系数 × 食堂系数
```

5 个子因子均为乘法关系。

| 比较好 | 1.1 |
| 中规中矩 | 1.0 |
| 比较坑 | 0.9 |
| 简直噩梦 | 0.8 |

**同事关系**：

| 选项 | 系数 |
|------|------|
| 亲如一家 | 1.2 |
| 和和睦睦 | 1.1 |
| 萍水相逢 | 1.0 |

| 选项 | 系数 | 说明 |

```
envFactor = 1.1 × 1.1 × 1.1 × 0.80 × 1.0 = 1.0648
```

### 3.3 expectedDailySalary — 期望日薪（核心创新）

```
期望年薪 = 基础年薪 × 城市系数 × 行业系数
期望日薪 = 期望年薪 / 260
```

这是应届生评测器的核心创新：用一个动态计算的"你应该拿多少"替代传统的固定常数基准。

### 3.4 effectiveHours — 有效工时

```
effectiveHours = workHours + commute × officeRatio × shuttleFactor - 0.5 × restTime
```

#### 3.4.1 各项参数

**workHours** — 日均工时（小时），用户直接输入。

**commute** — 通勤时间（小时），用户直接输入。

**officeRatio** — 办公室比例：
```
officeRatio = (每周工作天数 - WFH天数) / 每周工作天数
```
- WFH 天不需要通勤，按比例折算。
- 示例：每周 5 天，WFH 2 天 → `officeRatio = (5-2)/5 = 0.6`

**shuttleFactor** — 班车系数：

| 有无班车 | 系数 |
|----------|------|
| 无 | 1.0 |
| 有班车 | 0.3（通勤时间变为 30%） |

**restTime** — 休息时间（小时），每日工作中的午休/休息。

**0.5 × restTime** — 休息时间只有一半算作"付出"，因为这部分并非高强度劳动。

#### 3.4.2 完整示例

```
workHours = 9h
commute = 1h
每周 5 天, WFH 0 天 → officeRatio = (5-0)/5 = 1.0
无班车 → shuttleFactor = 1.0
restTime = 1.5h

effectiveHours = 9 + 1 × 1.0 × 1.0 - 0.5 × 1.5 = 9.25h
```

## 4. 评级体系

| 分值范围 | 评级 | 颜色（Tailwind） | 含义 |
|----------|------|------------------|------|
| < 0.5 | 大冤种 | `text-pink-800` | 严重低于市场，快跑 |
| 0.5 ~ 0.7 | 偏低 | `text-red-500` | 明显低于期望 |
| 0.7 ~ 0.9 | 一般 | `text-orange-500` | 略低于期望，可以谈 |
| 0.9 ~ 1.1 | 合理 | `text-blue-500` | 符合市场水平 |
| 1.1 ~ 1.3 | 不错 | `text-green-500` | 超出期望 |
| 1.3 ~ 1.6 | 很香 | `text-purple-500` | 远超期望 |
| ≥ 1.6 | 天选 Offer | `text-yellow-400` | 顶级 offer |

**评级边界说明**：
- Score = 1.0 是核心锚点，对应"刚好等于市场期望"
- 0.9~1.1 是"合理区间"，接受不会有明显亏欠感
- 向下区间（<0.9）更密集，因为"亏了"的感知更敏感
- 向上区间（>1.1）跨度更大，高薪 offer 稀缺

## 6. 完整计算示例（逐步演示）

### 场景：成都 985 本科，互联网公司 Offer

#### 输入参数

```
学历: 本科, 学校等级: 985/211
目标城市: 成都（新一线）, 目标行业: 互联网
年薪: ¥220,000
国家: 中国 (PPP=4.19)
每周工作: 5天, WFH: 0天
年假: 5天, 法定假日: 13天, 带薪病假: 3天
日均工时: 9h, 通勤: 1h, 休息: 1.5h
工作环境: CBD (1.1), 城市生活系数: 新一线 (0.80)
领导: 善解人意 (1.1), 同事: 和睦 (1.1)
无班车、无食堂
```

#### 第 ① 步：计算实际年工作日

```
年工作日 = 52 × 5 - (5 + 13 + 3 × 0.6)
         = 260 - 19.8
         = 240.2 天
```

#### 第 ② 步：计算实际标准化日薪

```
dailySalary = (220,000 × 4.19) / 240.2
            = ¥3,836.39
```

> **注**：此处 PPP 标准化的具体实现取决于项目的统一处理方式。如果 PPP 不直接乘入日薪而是做其他处理，需保持一致。

#### 第 ③ 步：计算环境系数

```
envFactor = 1.1 × 1.1 × 1.1 × 0.80 × 1.0 = 1.0648
```

#### 第 ④ 步：计算期望日薪

```
期望年薪 = 15万 × 1.0(成都/新一线) × 1.2(互联网) = 18万
期望日薪 = 180,000 / 260 = ¥692.31
```

#### 第 ⑤ 步：计算有效工时

```
officeRatio = (5 - 0) / 5 = 1.0
effectiveHours = 9 + 1 × 1.0 × 1.0 - 0.5 × 1.5
               = 9 + 1 - 0.75
               = 9.25h
```

#### 第 ⑥ 步：计算最终得分

```
Score = (dailySalary × envFactor × 8) / (expectedDailySalary × effectiveHours)
      = (3836.39 × 1.0648 × 8) / (692.31 × 9.25)
      = 32,666.52 / 6,403.87
      ≈ 5.10
```

> **注意**：以上数字包含了 PPP 标准化。如果项目中 PPP 不以这种方式参与计算（例如仅用于跨国显示），则应去掉 PPP 因子。以下是不含 PPP 的版本：

```
dailySalary = 220,000 / 240.2 = ¥915.90
Score = (915.90 × 1.0648 × 8) / (692.31 × 9.25)
      = 7,801.88 / 6,403.87
      ≈ 1.218 → 评级「不错」🟢
```

#### 第 ⑦ 步：对比不同工时

如果同样的 ¥22万 offer，但 996 工作制：

```
effectiveHours = 12 + 1 - 0.5 × 1 = 12.5h
Score = (915.90 × 1.0648 × 8) / (692.31 × 12.5)
      = 7,801.88 / 8,653.88
      ≈ 0.902 → 评级「一般」🟠
```

**结论**：同样是 ¥22万 offer，955 是"不错"(1.22)，996 是"一般"(0.90)。工时惩罚自然生效。

## 7. 公式推导历史（设计决策记录）

开发过程中经历了多次公式修正，记录在此以供理解设计意图。

### 7.1 第一版（已废弃）

```
Score = (dailySalary × envFactor) / (expectedDailySalary × effectiveHours × baseEnvFactor)
```

**问题**：Score ≈ 0.15，太低。因为期望日薪是"日薪"级别，再乘以 effectiveHours（约 9~12），分母过大。

### 7.2 第二版（已废弃）

```
Score = (dailySalary × envFactor) / (expectedDailySalary × baseEnvFactor)
```

**问题**：工时因素被完全忽略。同样是 ¥15万，955 和 996 算出相同分数。

### 7.3 最终版（当前采用）

```
Score = (dailySalary × envFactor × 8) / (expectedDailySalary × effectiveHours)
```

**解决方案**：分子乘以 8（标准工时），使工时惩罚自然生效。推导逻辑：

```
分子 = dailySalary × envFactor × 8
     = (年薪/年工作日) × 环境 × 8
     = "你的 8 标准工时的等效日收益"

分母 = expectedDailySalary × effectiveHours
     = (期望年薪/260) × 有效工时
     = "你背景对应的时薪 × 你实际付出的工时"
```

当 effectiveHours > 8 时，分母增长快于分子 → Score 下降 → 工时长 = 惩罚。

## 9. 边界情况与注意事项

### 9.1 边界情况

| 场景 | 处理方式 |
|------|----------|
| 学历=专科 | 基础年薪=6万，Score 可能偏高（低期望易达到） |
| 城市=三线 + 行业=教育 | 期望薪资极低（6×0.7×0.8=3.36万），Score 可能虚高 |
| 年薪=0 | 提示用户必须输入有效薪资 |
| WFH=全部天数 | 通勤=0，officeRatio=0，无通勤成本 |
| 食堂=丰富且便宜 + 班车=有 | envFactor 会显著偏高 |

### 9.2 与跳槽评测器 Score 的不可直接比较

应届生评测器的 Score 1.0 ≠ 跳槽评测器的 Score 1.0。两者公式结构不同，分母基准不同。不要在 UI 上直接并列对比两个 Score。

### 9.3 数据更新

期望薪资的基础年薪表具有时效性（校招薪资每年变化）。建议：
- 硬编码当前版本数据（2024-2025）
- 预留数据更新接口
- 每年 Q1 更新一次
